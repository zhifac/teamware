This is a GATE plug-in to allow you using DocServise as GATE DataStore.

How to use it:

You need only ./lib/* and creole.xml to run it as a GATE plug-in.

Create a directory under your plugins directory of GATE. For example "gate31/plugins/service-datastore"

Copy ./lib/* and creole.xml in this directory.

Run GATE and open "Plugins management console", check "service-datastore" plugin and press OK

In the GATE resource tree, right click on the "Data stores" and in the pop-up menu select "Open datastore"

In the window which allows you to select the type of datastore select "ServiceDataStore: document service storage using Web Services".
This option becames visible only if plugin "service-datastore" is loaded.
Note: If you loaded "service-datastore" but don't see a new data store type "ServiceDataStore: document service storage using Web Services" among the other 3 well known datastores (serial, oracle and postgress), so you need to get latest version of GATE from the repository (DataStoreRegister and MainFrame classes was changed)

In the next window you will be asked to enter document service URL. The basic form of URL is following: http://localhost:8080/docservice/services/docservice
You don't need to change anything if document service is deployed with defaults on your local PC, just press OK.
Substitute 'localhost' with the name of the server if the document service is deployed with defaults on other computer and then press OK.

If connection to the GLEAM document service was successfully established you will see a new data store in the GATE resource tree.

You can use this data store as any normal GATE data store.

How to build it:

There is build.xml for ant in the root directory of this project. Use 'build' target.
As result it will produce/update 'service-datastore.jar' in the ./lib directory.
